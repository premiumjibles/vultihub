Phase 1 / Weeks 1–4

Foundations

Every surface running on one shared SDK with reliable core wallet operations.

  • Unify the SDK, Consolidate all chain logic (EVM, Cosmos, Solana, Sui, TON, Tron, UTXO) into a single TypeScript package consumed by every product
  • Ship a fully functioning agentic wallet to Station, Hype migration screen first, then core wallet ops (balances, sends, swaps, receives) working reliably on every chain. No burnt pizza: we only go live when it's ready
  • Add UTXO chains, Bitcoin, Litecoin, Dogecoin, Bitcoin Cash via native wallet-core bindings
  • Begin Chrome extension, chat UI shell, vault connection. SDK exposed via API/MCP rather than bundled, so updates propagate automatically without OTA releases
  • Accelerate TSS integration, Groundwork already proved out. Begin implementing TSS flows in the agent app: multi-device relay coordination, session joining, timeout handling

Phase 2 / Weeks 5–8

  • Chrome extension live, Send, swap, chat with the AI agent directly from the browser toolbar

  • Staking support, Cosmos validators, Solana stake accounts, ETH staking across all surfaces

Phase 3 / Weeks 9–12

Autonomous Agents

The wallet evolves from a tool you operate into a system that acts on your behalf, within user-defined guardrails.

  • Agents as TSS signers, M-of-N signing landed in Phase 2. Now agents hold key shares and participate in signing sessions just like devices
  • Agent runtime, Containerized AI agents as persistent processes. One-click hosted deploy or self-hosted via Docker Compose
  • Configurable guardrails, Custom system prompts with enforced limits: max tx size, asset whitelists, daily spend caps, cool-down periods
  • Adversarial co-signing, Two agents with different mandates must both approve. Growth vs. risk agent debate protocol
  • Logged reasoning, Full audit trail of every debate, argument, and decision for user review and trust-building

Cadence

Delivery Rhythm

Lightweight coordination designed for speed. Automated visibility so engineers stay heads-down shipping.

  • 2-week mission cycles, work is shaped into goal-oriented missions with clear scope and timeboxes, assigned at the start of each cycle
  • Daily automated pulse, a bot summarizes each engineer's git activity (PRs, branches, areas touched) and posts to Discord. Zero manual reporting
  • Weekly 30-min sync, course corrections, architectural decisions, and blockers only. Not a status update
  • Self-test, self-merge, CI enforces quality (lint, types, tests under 2 minutes). Engineers ship without waiting on reviews. Critical areas like signing and key material require pre-merge review
  • Lightweight mission tracking, GitHub Issues for mission-level visibility and milestones. No heavyweight ticket management
