# Chat App: Scaffold standalone app and port agent UI

## Objective

Create a new standalone React application for the Vultisig AI agent chat, extracting the ~10K lines of agent UI from vultisig-windows into its own deployable app. This app is the frontend for the AI agent ‚Äî it renders conversations, handles user input, and displays transaction status. All crypto logic will come from @vultisig/sdk (wired in the next ticket).

## User Story

Users want a lightweight, web-deployable AI chat interface for their Vultisig wallet that doesn't require the full desktop app. The chat app talks to the agent backend and uses the SDK for wallet operations.

## Design Constraints

- React 19 + Vite (matching vultisig-windows stack)
- styled-components for styling (matches existing agent UI code)
- React Query for data fetching
- Must support web deployment (Vercel/similar) ‚Äî no Wails dependency
- The agent backend (`agent.vultisig.com`) remains unchanged ‚Äî this app is a new frontend for it
- i18n: can start English-only, but use react-i18next structure for future locales

**External dependencies to replace/abstract:**

1. **CoreStorage** ‚Äî vault/coin/address book data access ‚Üí will use SDK
2. **WalletCore provider** ‚Äî TrustWallet integration ‚Üí SDK handles internally
3. **Navigation** (`useCoreNavigate`, `CoreView`) ‚Äî replace with React Router
4. **Theme** (`getColor`, `ThemeColors`) ‚Äî port the color palette and theme provider
5. **i18n** ‚Äî port the English translations from `core/ui/i18n/en.ts` (agent-related keys only)
6. **Shared UI components** (`Button`, `Stack`, `Text`, `Modal`, `PageHeader`, etc.) ‚Äî port or replace with a small component library

**Routing in vultisig-windows:**
- `agent` view ‚Üí `AgentPage` (session list)
- `agentChat` view ‚Üí `AgentChatPage` (active conversation)
- The app navigates between these plus send/swap flows triggered by agent actions

## Gotchas

- The orchestrator currently creates tool handlers that import directly from `@core/chain/` and `@core/mpc/` ‚Äî in this ticket, stub those out or leave them broken. Ticket tic-XXXX (rewire to SDK) handles the actual wiring.
- `AgentContextService` requires injected functions (`getVault`, `getVaults`, `getVaultCoins`, `getAddressBookItems`) ‚Äî these will be wired to SDK in the next ticket, but the interface must be preserved.
- The shared UI components (`Button`, `Stack`, `Text`, `Modal`) are in `lib/ui/` which is an upstream mirror ‚Äî extract only what's needed, don't copy the entire lib.
- `styled-components` theme must be set up with a `ThemeProvider` ‚Äî the existing agent code expects `getColor()` and theme tokens to be available.
- The 23 icon components are simple SVGs ‚Äî straightforward to copy.
- `react-markdown` is used for `MarkdownContent.tsx` ‚Äî include it as a dependency.
