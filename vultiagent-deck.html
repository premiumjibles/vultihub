Why Watts, Not Dollars

Dollar amounts create micro-loss anxiety on every action. An abstracted currency (like Robux, airline miles, game gems) reduces spending friction while maintaining a real cost floor. Crypto users are financially sophisticated — they'll appreciate knowing 1W ≈ $0.01 without seeing "$0.25 deducted" every swap.

Watt Costs by Operation

Flash ops (chat, balance, send): 1W. Haiku ops (planning, crons): 3W. Sonnet ops (strategy, autonomous): 15W. First-time action: agent tells you the cost once, then never asks again.

100%

Basic ops continue on Flash (degraded)

Hard stop on all AI (emergency only)

Only if user is burning through degraded ops at extreme rate. Manual wallet functions in core app always available.

First-Time Action UX

First time a user triggers any action type, the agent says: "This is your first swap — just so you know, swaps use 3 Watts per operation. You have 197 remaining. Continue?" Then never asks again. Users can always check costs by asking the agent directly.

Open Questions for the Team

VULT Bonus: Tiers or Curve?

Option A (tier thresholds) is simpler and aligned with existing app. Option B (sliding scale) rewards any accumulation but adds complexity. Can we hybrid — tier unlocks + small continuous drip?

$2 Free Budget: Right Number?

200 W/mo lets casual users feel free but costs us ~$0.78/user/mo (after markup). For 8K free users = $6.2K/mo burn. Should it be $1.50? $1? Need utilization data.

Flash Tool Reliability

Can Gemini 2.5 Flash reliably execute our 22 MCP tools? Need to test top 10 flows for reliability & latency. If <95% success, cheap models save nothing if swaps fail.

Can We Afford Free Watts for VULT Holders?

Both options give Platinum+ holders free AI access. How many Platinum+ holders exist today? What's the real AI cost to subsidize them? At what holder count does it become unsustainable? Need data from core app.

Degradation vs Hard Stop

When free budget depletes: continue basic Flash ops (costs us, keeps user happy) or hard stop everything (saves money, risks churn)? Recommend: continue basic ops, gate Sonnet/Haiku.

Credit Pack Pricing

$2.49 for 200W / $5.99 for 500W / $10.99 for 1,000W. Large pack intentionally near sub price as upgrade gateway. Right markup? Too high risks churn, too low and nobody subscribes.

New Mission

Billing & Token Gating Infrastructure

Everything needed to ship Watt budgets. Validates the pricing model with real data, then enforces it.

1. Flash Reliability Testing

Run top 10 tool flows against Gemini 2.5 Flash. Measure success rate and latency per operation. Target >95%. If Flash can't reliably execute our MCP tools, the cost model doesn't work.

2. Measure Token Cost per Operation

Instrument the agent to log LLM token counts per action type. Calculate real per-op costs across Flash, Haiku, Sonnet. Map each operation to its Watt price.

3. Keyword Pre-Router

Regex/keyword classifier for obvious intents (balance check, simple send). Routes directly to Flash, skipping the Haiku classification call on 80%+ of traffic.

4. Token Gating System

Track per-user Watt balance. Deduct on each operation. Enforce tier access — gate Haiku/Sonnet features when budget depleted. Graceful degradation to Flash. Credit pack purchase flow.

Steps 1–2 are prerequisites — we need real cost data before committing to Watt prices. Steps 3–4 are the billing system itself.
